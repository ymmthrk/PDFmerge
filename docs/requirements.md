# PDFmerge 要件定義書

**バージョン**: 1.4.0
**最終更新日**: 2026-01-29
**ステータス**: 追加機能実装完了

## 1. プロジェクト概要

### 1.1 目的と背景

業務で複数のPDFファイルを結合する際、既存の無料Webアプリでは外部サーバーにファイルをアップロードする必要があり、機密情報の漏洩リスクが存在する。本ツールは完全にローカルで動作し、外部通信を一切行わないことで、情報漏洩リスクをゼロにしたPDF結合機能を提供する。

### 1.2 ターゲットユーザー

- 業務で機密文書を扱うビジネスユーザー（個人利用）

### 1.3 核となる機能と価値

| 機能 | 提供する価値 | この機能がないと |
|------|-------------|----------------|
| PDF結合 | 複数PDFを1つのファイルに統合 | プロジェクトの目的を達成できない |
| ファイル順序操作 | ドラッグ&ドロップで直感的に並び替え | 意図した順序で結合できない |
| ページ操作 | 回転・順序変更・抽出で細かな調整 | 不要ページの除外や向き修正ができない |
| ローカル動作 | 外部通信なし、情報漏洩リスクゼロ | 既存Webアプリと差別化できない |

## 2. 機能要件

### 2.1 実装済み機能

#### F-001: PDFファイル結合 ✓
- 複数のPDFファイルを1つのPDFに結合する
- ファイル数・サイズの上限は設けない（メモリ不足時はエラー表示）
- 結合順序はユーザーが指定した順序に従う
- 結合後のファイル名と保存先をユーザーが指定できる
- 結合処理中はプログレス表示し、キャンセル可能

#### F-002: ファイル追加 ✓
- ファイル選択ダイアログからPDFを追加できる（複数選択可能）
- ドラッグ&ドロップでPDFを追加できる（複数ファイル同時可）
- PDF以外のファイルは受け付けない（エラー表示）
- 破損したPDFはエラー表示して追加しない
- 複数ファイル追加時に一部が破損/非対応の場合、有効なファイルのみ追加し、問題のあるファイルは個別にエラー表示
- 同じファイルを重複追加しようとした場合は警告して拒否

#### F-003: ファイル順序操作 ✓
- ドラッグ&ドロップでファイルの順序を変更できる
- 上へ/下へボタンでも順序変更できる
- ファイルリストから個別に削除できる
- クリアボタンで全ファイル削除時は確認ダイアログを表示

#### F-004: ファイルリスト表示 ✓
- 追加されたファイルを一覧表示する
- 表示項目：ファイル名、ファイルサイズ、ページ数
- 現在の順序番号を表示する
- 回転・順序変更・抽出設定のアイコン表示（🔄📄✂）

#### F-005: ページ回転 ✓
- 指定ページを90°/180°/270°回転できる
- 対象選択：全ページまたは範囲指定（例: 1-3, 5, 7-10）
- 回転リセット機能：回転を0°に戻す

#### F-006: ページ順序入れ替え ✓
- 1ファイル内のページ順序を変更できる
- ドラッグ&ドロップまたは上下ボタンで操作
- 逆順・リセット機能

#### F-007: ページ抽出 ✓
- 特定ページのみを結合対象にできる
- 範囲指定（例: 1-3, 5, 7-10）または個別選択
- 奇数ページ・偶数ページ一括選択

#### F-008: サムネイルプレビュー ✓
- 選択ファイルの全ページをサムネイル表示
- 回転・順序変更・抽出設定を反映した表示
- スクロール可能なリスト形式

#### F-009: 結合プレビュー ✓
- 結合後のPDFを保存前に確認できる
- ページ送り機能（前へ/次へ、ページ指定）
- プレビューから直接保存可能

#### F-010: バージョン管理 ✓
- ウィンドウタイトルにバージョン表示
- Aboutダイアログでバージョン・著作権情報表示
- exe ファイルのプロパティにバージョン情報埋め込み

#### 状態と動作

| 状態 | 動作 |
|------|------|
| 初期状態 | ファイルリスト空、結合ボタン無効 |
| ファイル追加時 | リストに追加、順序番号自動付与、結合ボタン有効化 |
| 複数ファイル追加時（一部エラー） | 有効ファイルのみ追加、問題ファイルは個別にエラー表示 |
| ファイル1つのみ | 結合ボタン無効（2つ以上必要） |
| 重複ファイル追加時 | 警告ダイアログ表示、追加しない |
| 破損PDF追加時 | エラーダイアログ表示、追加しない |
| クリアボタン押下時 | 確認ダイアログ表示、OK時に全削除 |
| 結合実行時 | プログレスダイアログ表示（キャンセル可能）、完了後に成功メッセージ |
| エラー時 | エラーダイアログ表示（原因を明記） |

## 5. 非機能要件

### 5.1 セキュリティ

- **完全オフライン動作**: 外部ネットワーク通信を一切行わない
- **ファイルアクセス制限**: ユーザーが明示的に選択したファイルのみアクセス

### 5.2 パフォーマンス

- 通常の業務文書（数十〜数百ページ）の結合は数秒以内で完了
- 大量・大容量ファイルの場合はプログレス表示で進捗を可視化

### 5.3 配布形態

| 版 | ファイル | サイズ |
|----|---------|--------|
| メイン版（PySide6） | PDFmerge.exe | 約70MB |
| 軽量版（tkinter） | PDFmerge_tk.exe | 約26MB |

- 単一exeファイルとして配布
- インストーラー不要
- 依存ランタイム不要（Python不要で実行可能）

### バージョン番号管理

- **ファイル**: `src/version.py`（一元管理）
- **形式**: セマンティックバージョニング (MAJOR.MINOR.PATCH)
  - MAJOR: 後方互換性のない変更
  - MINOR: 後方互換性のある機能追加
  - PATCH: バグ修正

### バージョン更新手順

1. `src/version.py` を編集
   ```python
   VERSION = "1.0.1"
   VERSION_TUPLE = (1, 0, 1, 0)
   ```

2. `version_info.txt` を編集（Windows exeプロパティ用）
   ```
   filevers=(1, 0, 1, 0),
   prodvers=(1, 0, 1, 0),
   StringStruct(u'FileVersion', u'1.0.1.0'),
   StringStruct(u'ProductVersion', u'1.0.1.0')
   ```

3. 再ビルド実行
