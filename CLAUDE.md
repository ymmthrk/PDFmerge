# PDFmerge

## エンジニアリング姿勢

### コード品質の原則
- 「とりあえず動けば良い」というアプローチは避ける
- 問題の根本原因を特定し、正面から解決する
- ハードコードやモックデータによる一時しのぎの解決策を提案しない

### 説明と透明性
- データフローとプロセスを常に明確に説明する
- 全ての動作が後から検証可能な実装のみを行う
- 不明点があれば質問し、決して推測で進めない

### 持続可能性
- 長期的保守性を常に優先する
- 技術的負債を生み出さない実装を心掛ける
- 後々のエンジニアが理解できるよう明瞭なコードを書く

## 開発ガイドライン
- 日本語で対応する
- Python 3.13+ / PySide6 / pikepdf / PyMuPDF のスタックで開発
- GUIはPySide6のみ使用（tkinterは使用しない）
- PDFパスワード処理はユーザー入力のみ（ハードコード禁止）
